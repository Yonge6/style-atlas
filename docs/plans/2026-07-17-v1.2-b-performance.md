# V1.2-B Performance Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Reduce unnecessary image work and make export/share operations memory-safe without changing the visual design, product behavior, access tiers, or release metadata.

**Architecture:** Keep the existing static HTML/CSS/JavaScript application and three-card deck. Add three bounded coordinators inside `game.js`: an image decode pipeline, a cancelable Wiki request manager, and a single-flight export state machine. Keep Native sharing single-flight in `WebViewBridge.swift`, with temporary files owned and cleaned by one operation lifecycle.

**Tech Stack:** Vanilla JavaScript, CSS, Playwright, SwiftUI, WKWebView, UIKit share sheet, StoreKit 2.
